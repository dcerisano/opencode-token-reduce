# task_completion — opencode-token-reduce

## No Standard Verification Commands
This repo has no linter, formatter, test runner, type checker, or build system. It is a configuration template, not a code project.

## What to Do After Changes
1. Validate `opencode.json` is valid JSON (no trailing commas, proper quoting).
2. Validate YAML frontmatter in agent and skill files (correct `---` delimiters, no stray syntax).
3. Run `git status` to check what changed.
4. Run `git diff` to review changes before committing.
5. If modifying `.opencode/skills/` files, ensure `name` in frontmatter matches the directory name.
