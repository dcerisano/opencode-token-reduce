# MCP Multi-Agent Framework — Core

## Source of Truth
- `tools/multi_agent.py` — single Python model defining agents, graph topology, tools, proto schema, and tests
- `tools/langgraph_to_proto.py` — AST-based Python→Java code generator (not template-based)
- All agent node wrappers, proto schemas, `GraphRunner.java`, `McpProtoTest.java`, and `McpIntegrationTest.java` are auto-generated from the Python model — manual edits are overwritten on rebuild

## Manual Code (editable)
- `src/main/java/research/v1/impl/*.java` — MCP server business logic
- `src/main/java/research/v1/*.java` — auto-generated by codegen: `OllamaClient.java`, `OllamaMCPClientImpl.java`, `AgentIdInterceptor.java`, `MCPLogger.java`, `AgentState.java`, agent nodes, `GraphRunner.java`, server mains, MCP server impls
- `src/main/java/research/v1/*.java` — manually written: `WebSearchTool.java`, `FilesystemTool.java`, `BaseAgentNode.java`, HTTP handlers, main entrypoints for HTTP servers
- `src/test/java/research/v1/McpUnitTest.java`, `McpHttpTransportTest.java` — manually written tests
- `tools/*.py` (except generated output) — code generation tools

## LLM Stub Mode (Testing)
- `OllamaClient.call()` checks system property `ollama.stub` at the top of every call
- If set to a non-empty string, returns that string immediately without any HTTP request
- Used by `McpHttpTransportTest` to avoid depending on a running llama-server
- All 25 HTTP transport tests run in ~1.6s (down from ~11s+ when waiting for LLM timeouts)
- Framework-level: works for any workflow or test that uses OllamaClient

## LLM Efficiency
- `max_tokens` from Python `_sample(system, user, max_tokens=N, ...)` is now propagated to generated Java — workflow authors' token limits are honored
- `BaseAgentNode.createMessage()` default fallback: 2048 (was 4096)
- `OllamaClient`: 10s connect timeout, 2 retry attempts (was 3), 500ms base delay (was 1000ms)

## Build
- ONE script: `./build.sh` — detects inside-Docker via `/.dockerenv`
  - Host side: GPU detection, llama-server check, model downloads, `docker run ... mcp-build`
  - Container side: clean, generate, compile (javac), start MCP servers, unit tests, integration tests
- `build-inner.sh` DELETED (merged into build.sh)
- ALWAYS run from Docker: `./build.sh` on host → Docker container → build steps inside
- Never run build directly on host

## Logging
- `output/integration-message.log` — final integration test log (moved from `message.log`)
- MCPLogger methods: `agentToAgent`, `agentToLlm`, `llmToAgent`, `toolToAgentSampling`, `agentToToolSampling`, `agentToTool`, `toolToAgent`
- HTTP handlers log tool→agent sampling via `toolToAgentSampling("CreateMessage")` / `agentToToolSampling("CreateMessageResult")`
- Orchestrator final `Complete` logged in GraphRunner when supervisor reaches `__end__`

## Architecture
- **Dual transport**: gRPC and JSON-RPC 2.0 over HTTP. MCP servers serve both; agent nodes try JSON-RPC HTTP first, fall back to gRPC
- Distributed agent nodes — each agent is independent gRPC server, individually deployable/testable
- Bidirectional MCP — tools callback to agents for LLM sampling (HTTP path: handler logs TOOL→AGENT Sampling)
- Deterministic supervisor routing (state-based if/else). LLM generates reasoning text only — routing decisions are rule-based, not LLM-driven
- Proto `Content` model uses oneof (TextContent, ImageContent, ResourceContent)

## JSON-RPC 2.0 Transport
- Hand-implemented (Gson + `com.sun.net.httpserver.HttpServer`), no third-party JSON-RPC library
- `McpHttpHandler.java` — abstract base handler with JSON-RPC routing, validation, notifications (no `id` → 202)
- `WebSearchMCPHttpHandler.java` (name: `web-search-mcp-server`), `FilesystemMCPHttpHandler.java` (name: `filesystem-mcp-server`)
- HTTP entry points: `WebSearchHttpMain.java` (port 50061), `FilesystemHttpMain.java` (port 50062)
- Agent node JSON-RPC clients: `SupervisorNode`, `ResearchAgentNode`, `WriterAgentNode` — each has `initHttpServer()` + `jsonRpcRequest()`
- Tests: `McpHttpTransportTest.java` — validates version enforcement, notifications, parse errors
- HTTP ports = gRPC ports + 10 (enforced by code generator)
